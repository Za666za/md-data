# 上下文工程

- 决定将哪些信息放入上下文窗口
- 为什么不能把所有历史信息都给模型?
  - 一方面模型需要重新分析历史信息, 消耗token
  - 另一方面历史信息可能互相冲突, 可能已经过期,需要处理
  - 这就要求给模型构建人类一样的基于系统, 有重点,有过滤,有长期保留
- Harness: 用框架来控制模型输出安全性
  - 必要时可将结果最后留给人工审核

## 题

- 1.Agent和普通聊天机器人最大的区别是什么？
  - 解析:tool,自主规划, react
- 2.为什么大模型会出现“忘记”和“幻觉”?
  - 答案: 
    - "忘记":上下文过长信息丢失, 或设计时没有将历史拼入问题上下文
    - 幻觉: 训练时相关领域数据缺少; 模型历史训练知识过期已不适用 
- 3.个好的Prompt 少应该包含哪些信息?
  - 答案: 角色,背景,目标, 约束, 格式
- 4.RAG为什么要先分块再向量化？ 
  - 根据语义分块
  - 分块可以更精准的匹配问题, 提高知识的召回率(更容易找到)
- 如果上下文太长，可能带来哪些问题？
  - 重要信息遗忘, 抓不到重点
  - 分析消耗过大
